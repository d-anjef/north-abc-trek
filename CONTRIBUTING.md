# Contributing to North ABC Expedition

Thank you for your interest in this project! 🏔️

## How You Can Help

### 🐛 Report Bugs
Found something broken? [Open an issue](https://github.com/YOUR_USERNAME/north-abc-expedition/issues/new) with:
- Clear description of the bug
- Steps to reproduce
- Screenshots if applicable
- Browser/device info

### 💡 Suggest Features
Have an idea? Open an issue tagged \`enhancement\`.

### 🔀 Submit Pull Requests

1. **Fork** the repository
2. **Clone** your fork: \`git clone https://github.com/YOUR_USERNAME/north-abc-expedition.git\`
3. **Create a branch**: \`git checkout -b feature/your-feature-name\`
4. **Make your changes**
5. **Test locally**: \`npm run dev\`
6. **Build successfully**: \`npm run build\`
7. **Commit**: \`git commit -m "feat: your descriptive message"\`
8. **Push**: \`git push origin feature/your-feature-name\`
9. **Open a Pull Request**

## Commit Message Convention

Use conventional commits:
- \`feat:\` — New feature
- \`fix:\` — Bug fix
- \`docs:\` — Documentation changes
- \`style:\` — Formatting (no code change)
- \`refactor:\` — Code restructuring
- \`perf:\` — Performance improvement
- \`test:\` — Adding tests
- \`chore:\` — Maintenance

## Code Style

- Use consistent indentation (2 spaces)
- Keep components focused and small
- Comment complex logic
- Use semantic HTML
- Follow existing CSS naming conventions (BEM-ish)

## Building Your Own Trek Story

If you want to use this as a template for YOUR trek:

1. Fork this repo
2. Replace all photos in \`public/images/\`
3. Update data files in \`src/data/\`
4. Customize colors in \`src/index.css\`
5. Deploy to Vercel
